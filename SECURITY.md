# Security Policy

## Supported Versions

rucher is pre-1.0 and under active development. Security fixes land on `main`
and in the latest tagged release only — there are no backports to earlier
releases. Run a recent build.

## Reporting a Vulnerability

Please report security issues **privately**. Do not open a public issue, pull
request, or discussion for anything security-sensitive.

Preferred: use GitHub's private vulnerability reporting — open the repository's
**Security** tab and choose **Report a vulnerability**
(<https://github.com/igk1972/rucher/security/advisories/new>).

Alternatively, email **igk@igk.one**. Please include:

- a description of the issue and its impact,
- steps to reproduce (a minimal proof of concept if possible),
- the affected version or commit, and
- any suggested remediation.

## What to expect

- Acknowledgement of your report within a few days.
- An assessment and, for confirmed issues, a fix on `main` and a new release.
- Coordinated disclosure: we'll agree on a timeline before publishing details,
  and credit you in the advisory unless you prefer to stay anonymous.

Thank you for helping keep rucher and its users safe.
